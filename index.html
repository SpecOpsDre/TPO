📊 GTM Training – By Frank Savidge

✅ Concise Summary

  • GTM Purpose: Manage tracking tags without altering website code directly.
  • Core Concepts:
    • Tag: Sends data (e.g., to Floodlight, Meta).
    • Trigger: Specifies when a tag fires.
    • Variable: Supplies dynamic data (e.g., revenue, product ID).
  • Tag Creation: Use templates or custom HTML snippets.
  • Data Layer: Contains dynamic values used in tags/triggers.
  • Testing & Debugging:
    • Use GTM Preview mode and Chrome DevTools.
    • Verify tag firing and variable population.
  • Common Issues: Poor site structure and delayed client responses.
  • Client Notes: Smaller clients are more hands-on; larger ones more delayed.

📝 Detailed Summary

🧠 Introduction & Terminology

  • GTM allows deployment of tracking tags without modifying source code.
  • Site tag: Tracks user activity on a website.
  • Tag: Contains code snippet + trigger + variable.

🧰 Building a Tag in GTM

  1. Start with the tag:
    • Use template (e.g., Floodlight, Meta Pixel) or custom HTML.
    • Example: Floodlight needs tag string and activity string.
  2. Add variables:
    • Identify needed data: revenue, product ID, etc.
    • Use data layer (e.g., ecommerce.current.items.price).
  3. Create triggers:
    • Specify when tag should fire (e.g., on purchase confirmation).
    • Custom events are more reliable than URL-based triggers.

🧪 Testing & Validation

  • Use GTM Preview Mode to test tag firing and variable data.
  • Use Chrome DevTools → Network tab to confirm tag success and platform delivery.

⚠️ Common Problems

  • Badly structured data layers make finding variables difficult.
  • Clients not providing specs upfront cause back-and-forth delays.
  • Clients often avoid doing test purchases, even when required.

👥 Client Scenarios

  • Smaller clients (especially in Scotland) are more cooperative with tagging.
  • Larger brands (e.g., Tesco, Sky) often handle tagging internally or via agencies, which slows implementation.

🧪 Final Demo

  • Meta Pixel tag example shown:
    • Used Meta template and inserted Pixel ID.
    • Trigger set to fire on All Page Views, no variable or data layer needed.

🔍 MX Training - By Andre Ferreira

✅ Concise Summary

  • 🔧 Purpose of Training: Help OPS and Tech teams understand MX and its applications.
  • 📌 Core Concepts: MX is for data visualization only; all campaign changes happen in Prisma.
  • 🛠️ Main Functions: Installation, navigation, querying, exporting, unlocking, and troubleshooting.
  • 📤 Templates & Exports: Save time with reusable field templates for specific needs.
  • 🧩 Extra Tools: Unlock Wizard for single-line unlocks with additional training available.

📝 Detailed Summary

🔧 Purpose of Training

  • Help OPS and Tech teams understand what MX is and how it's used.
  • Covers basic and advanced features: installation, navigation, querying, exporting, unlocking, and troubleshooting.
  • Intended to supplement current workflows and clarify MX's limitations and integration with Prisma.

📌 Core Concepts

  • MX is for visualizing data only; all campaign edits/bookings happen in Prisma, not MX.
  • MX updates in real-time when changes are approved in Prisma and data is refreshed.

🛠️ Main Functions Covered

  1. Installing, Repairing, Updating MX
  2. Navigating Queries:
    • Filter reports (e.g., by client or date).
    • Identify relevant campaigns using IOs.
  3. Validating Data:
    • Compare invoice data against campaign details in MX/Prisma.
    • Ensure invoice matches are accurate (e.g., gross vs. payable amounts).
  4. Exporting Data:
    • Export reports to Excel after applying filters and templates.
    • Use templates to customize columns for specific audit or finance needs.
  5. Unlocking Lines:
    • Access via lock icon.
    • Right-click to insert unlock requests for editing fields like commission, value, or discounts.
    • Use Unlock Wizard for single-line unlocks.

📤 Templates & Exports

  • Templates help save time by reusing specific fields.
  • Only filtered data is exported; large datasets are not feasible for full export.

🧩 Extra Tools

  • Unlock Wizard for single-line unlocks.
  • Further MX Unlock training available for more complex workflows.

🚦 Trafficking Training - By Lauren Pringle

✅ Concise Summary

  • 🎯 Purpose: Ensures accurate data collection, consistent metrics, and error detection.
  • 🔧 Best Practices: Ask the right questions early, collaborate with teams, understand different media setups.
  • 🏷️ Tag Types: 1x1 tags, click tags, event pixels, URL swaps, DCO, video tags.
  • 🎬 Creative Rotation: Even, optimized, sequential, and random options.
  • 💡 Tip: Trafficking becomes easier with experience – don't hesitate to ask questions!

📝 Detailed Summary

🎯 Purpose & Importance

  • Trafficking plays a critical role in digital advertising by ensuring:
    • 📊 Accurate data collection (impressions, clicks, conversions, user behavior)
    • 🧮 Consistency between platform and third-party metrics
    • 🚨 Quick error detection (e.g., pixel misfires, delivery discrepancies)
    • 🛡️ Compliance and audit readiness (e.g., GDPR, CCPA, SLAs)
    • 🧾 Invoice reconciliation (independent tracking for cost verification)
    • 📈 Real-time performance reporting and optimization opportunities

🔧 Foundational Best Practices

  • ❓ Ask the right questions early (e.g., platform setup, pixel needs, creative hosting)
  • 🤝 Collaborate closely with planning teams and suppliers
  • 🔍 Understand the trafficking setup for different media types and buying paths (PST vs. direct)

🏷️ Tag Types & Key Concepts

  • 📌 1x1 Tags
    • Lightweight image pixels used when heavier tracking is restricted
    • Common in:
      • 📺 Platforms with strict tracker policies (e.g., YouTube via DV360)
      • 📱 Environments like in-app or social media
    • ✅ Always monitoring tags, ❌ never blocking
    • 🚫 Cannot be associated directly with creatives
  • 🔗 Click Tags
    • Always 1x1 tags
    • Only click tracking is implemented (no impression tracking)
  • 🎯 Event Pixels
    • Used to track specific actions like conversions or page views
    • Can be embedded in main tags or triggered by user interaction
    • ⚠️ May conflict with Campaign Manager's radar pixel (removal often required)
  • 🔁 URL Swaps
    • Require full trafficking sheet updates, not just shared links
    • ✅ Always verify URLs before submission to avoid campaign issues
  • 🎨 DCO (Dynamic Creative Optimization)
    • Real-time creative personalization
    • Requires:
      • 🧱 DCO base file (template from creative agency)
      • 🧩 DTK (Dynamic Targeting Key) for targeting and logic control

🔄 Creative Rotation in CM

  • Controls how multiple creatives are served:
    • ⚖️ Even: Equal rotation
    • 🚀 Optimized: Based on performance
    • 🧵 Sequential: Fixed order
    • 🎲 Random: No pattern

🎬 Video Tags

  • Preferably hosted when possible
  • Key checks:
    • 🧠 Supplier acceptance of VPAID (interactive ads)
    • 👁️ OMID support (viewability measurement)
    • 🛠️ IAS wrapping compatibility
  • 🧪 Alternatives:
    • Request supplier VAST tag for wrapping
    • Use UVP tag or Non-Standard Tag templates if wrapping is not supported

💡 Final Note

  • Trafficking can be complex at first, but becomes easier with experience.
  • 🙋 No question is a stupid question — reach out to your team for support anytime!
